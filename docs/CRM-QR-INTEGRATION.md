# Bententrade — QR, сайт и доработка CRM

Документ для команды CRM и бэкенда. Описывает **что добавить в CRM**, **зачем**, **какие API нужны сайту** и **как устроены сценарии**. Реализация на сайте идёт после согласования этой спецификации.

---

## 1. Цель интеграции

На каждый моток искусственного ротанга клеятся **два QR-кода** (печатаются **в CRM**). Оба ведут на сайт по одному формату URL:

```
https://bententrade.uz/r/{qr_token}
```

Пример: `https://bententrade.uz/r/k4f82ms`

### Два типа QR (согласовано)

| Тип | `qr_type` | Назначение после скана |
|-----|-----------|-------------------------|
| **QR отзыва** | `review` | Оставить отзыв на bententrade.uz; отзыв **дублируется** на сайт компаньона (партнёра) |
| **QR каталога** | `catalog` | Переход в каталог искусственного ротанга на bententrade.uz |

Тип задаётся в CRM при генерации/печати бирки. Сайт по `qr_token` получает `qr_type` и открывает нужный сценарий (лендинг отзыва или редирект/лендинг каталога).

**CRM полностью управляет сайтом** в части:

- печати и учёта QR;
- каталога товаров (добавление, цены, фото, публикация — только из CRM);
- аналитики всех действий с устройства;
- маршрутизации заявок и дублирования отзывов партнёру.

Сайт — витрина и API; **источник правды по товарам и QR — только CRM**.

Сайт должен:

1. Распознать `qr_token` и `qr_type`.
2. Записать сканирование и **все** действия пользователя (карточка, корзина, каталог…).
3. Требовать регистрацию (телефон) для отзыва, корзины, заявки, вопросов.
4. Скрыто закрепить пользователя за дилером (если QR привязан).
5. Отправлять события и сущности в CRM через webhooks.
6. Для QR отзыва — после публикации отправить копию отзыва партнёру.

**Номер партии (`batch_code`) — только в CRM, на сайте не показывается никогда.**

**Критично:** старые QR-ссылки не должны ломаться. `qr_token` постоянный; меняются тип, статус и привязки, не URL.

**Приватность (обязательно):** на сайте пользователь **никогда не видит**, что он «покупатель дилера X» или привязан к конкретному партнёру. Имя дилера, `dealer_id`, маршрутизация заявок — **только бэкенд и CRM**. На лендинге QR и в личном кабинете показывается только бренд **Bententrade** и данные о мотке/товаре.

### 1.1. Cookie consent (реализовано на сайте)

До принятия cookie пользователем **запрещена любая передача данных на сервер**. При этом **каталог** (`catalog.html`, `product.html`) и **личный кабинет** (`account.html`, `login.html`) уже реализованы и остаются доступны — с локальными/встроенными данными и `localStorage`.

| До согласия | После «Принять» |
|-------------|-----------------|
| Блокируются все запросы к `/api/*` и `/data/*` | Заказы, формы, вход/регистрация, sync каталога с CRM |
| Каталог и карточки товара — встроенный фолбэк + `products.js` | Актуальные цены, фото и товары из API (`catalog-sync.js`) |
| Кабинет: корзина и избранное только в браузере | Профиль, заказы, адреса, избранное с сервера (`account.js`, `auth.js`) |
| Корзина/checkout не отправляет заказ на сервер | `POST /api/orders` и остальные API |

Реализация: `assets/cookies.js` (баннер), `assets/api.js` (gate), страница `cookies.html`. Ключ согласия: `localStorage.btt_cookie_consent = accepted`. Событие `btt:cookies-accepted` перезагружает каталог, поиск и статьи.

## 1.1. Архитектура: две БД + webhooks (согласовано)

| | Сайт (Bententrade) | CRM (отдельный сервис) |
|---|-------------------|------------------------|
| **БД** | Своя PostgreSQL | Своя БД |
| **Зачем раздельно** | Каталог, сессии, корзина, публичный трафик QR | Дилеры, QR (мастер-данные), воронка, модерация, кабинеты |
| **Связь** | Сайт читает QR и каталог из CRM API; пишет события webhooks | CRM печатает QR, каталог, аналитика, дублирование отзывов партнёру |

```
Пользователь → bententrade.uz
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
   Сайт БД                 CRM БД
   (users, cart,           (qr_codes, products,
    orders, cache)          dealers, events,
        ▲                   reviews, печать QR)
        │                        │
        ├── webhooks (события) ──┤
        └── sync (каталог) ◄─────┘
              CRM → сайт
```

**CRM — источник правды** для: `qr_codes`, `dealers`, **каталога товаров**, статусов QR, аналитики, печати бирок.

**Синхронизация каталога (CRM → сайт):** при создании/изменении товара в CRM — webhook `POST {SITE}/api/sync/product` (или batch `catalog/sync`). Сайт **не редактирует** каталог в своей админке — только отображает то, что пришло из CRM.

## 3. Что добавить в CRM — сущности и зачем

### 3.1. Дилеры (`dealers`)

**Зачем:** внутренняя маршрутизация заявок и отзывов; кабинет дилера; аналитика по QR. **На сайте имя дилера не показывается.**

> **Про «список дилеров MVP»:** это не просьба прислать список заранее. Имелось в виду: *сколько дилеров завести в CRM перед первым запуском QR*. Ответ: **столько, сколько реально есть партнёров** — хоть один тестовый для проверки, остальных добавляете в CRM по мере печати бирок. Сайту не нужен фиксированный список; он получает `dealer_id` из ответа CRM по `qr_token`.

| Поле | Тип | Обязательно | Зачем |
|------|-----|-------------|-------|
| `id` | int/uuid | да | PK |
| `code` | string | да | Короткий код (`emil_kg`, `almaty_kz`) |
| `name` | string | да | Имя для админки («Эмиль») |
| `country` | enum | да | `UZ`, `KG`, `KZ`, `RU`, `TJ`, `OTHER` |
| `phone` | string | нет | Для кнопки «Позвонить» |
| `telegram` | string | нет | Username или ссылка |
| `whatsapp` | string | нет | Номер для wa.me |
| `email` | string | нет | Дублирование заявок |
| `is_active` | bool | да | Выключенный дилер не получает новые лиды |
| `created_at` | datetime | да | Аудит |

**Правило маршрутизации заявок:**

- Пользователь пришёл по QR дилера → заявки идут **дилеру + Bententrade** (видимость у обоих).
- Пользователь выбрал страну с активным дилером, но без QR → заявка **только Bententrade** (или по правилу страны — настраивается).
- Страна без дилера → **только Bententrade**.

### 3.2. QR-коды (`qr_codes`)

**Зачем:** постоянная ссылка; тип сценария (отзыв / каталог); печать бирки из CRM; аналитика воронки.

**Печать:** QR генерируется и печатается **в интерфейсе CRM** (PDF/PNG бирки с URL `bententrade.uz/r/{token}`). Импорт CSV — опционально, не основной путь.

| Поле | Тип | Обязательно | Зачем |
|------|-----|-------------|-------|
| `id` | int/uuid | да | PK |
| `token` | string, unique | да | Публичный `qr_token` в URL |
| `qr_type` | enum | да | **`review`** — отзыв; **`catalog`** — каталог ротанга |
| `status` | enum | да | `active`, `inactive`, `revoked` |
| `dealer_id` | FK → dealers | нет | Маршрутизация (скрыто на сайте) |
| `country` | enum | нет | Внутреннее |
| `motok_number` | string | нет | Номер мотка — можно показать на сайте |
| `batch_code` | string | нет | **Только CRM, на сайт не отдаётся** |
| `profile_id` | string | нет | Профиль ротанга |
| `profile_name` | string | нет | «Полумесяц 10 мм» |
| `color_id` | string | нет | Цвет |
| `color_name` | string | нет | Tobacco, Woody… |
| `weight_kg` | decimal | нет | Вес мотка |
| `product_id` | string | нет | Связь с карточкой каталога (sync из CRM) |
| `partner_id` | FK | нет | Для QR отзыва — куда дублировать отзыв |
| `label_template` | string | нет | Шаблон печати бирки |
| `printed_at` | datetime | нет | Когда напечатали в CRM |
| `notes` | text | нет | Внутренние заметки |
| `created_at` | datetime | да | Аудит |

**Поведение по типу:**

| `qr_type` | Сайт после скана |
|-----------|------------------|
| `review` | Лендинг «Настоящий ротанг Bententrade» + CTA «Оставить отзыв»; опционально каталог |
| `catalog` | Переход в `/catalog?cat=rattan` (с сохранением `qr_token` в сессии) |

**Партия:** поле `batch_code` хранится для производства и отчётов в CRM. API для сайта и публичный JSON **не включают** `batch_code`.

**Важно для обратной совместимости:**

- `token` **никогда не переиспользуется** и не меняется.
- При смене дилера/партии обновляются FK-поля, URL остаётся тем же.
- `status = revoked` — страница «QR недействителен», но HTTP 200 (не 404).

### 3.3. Пользователи — расширение (`users`)

**Зачем:** регистрация по телефону, закрепление за дилером (скрыто), личный кабинет.

**Обязательное поле при регистрации — только `phone`.** Без SMS-подтверждения, без пароля, без email в MVP.

**Модель входа (согласовано):**

1. Пользователь вводит телефон (+ имя/фамилия/страна по желанию).
2. Сайт проверяет только формат номера (мин. 7 цифр).
3. Сразу создаётся сессия (cookie) — **без OTP и без SMS**.
4. Повторный визит: тот же телефон → вход без кода (пока сессия жива или «войти по номеру» без SMS).

> SMS-верификация **не планируется** в первой версии. При злоупотреблениях — rate limit по IP и по номеру.

| Поле | Тип | Обязательно | Зачем |
|------|-----|-------------|-------|
| `phone` | string, unique | **да** | Логин и связь |
| `first_name` | string | нет* | *Рекомендуется собирать в форме |
| `last_name` | string | нет* | |
| `country` | enum | нет* | UZ, KG, KZ, RU, TJ, OTHER — влияет на UI |
| `email` | string | нет | Не показывать в форме MVP |
| `dealer_id` | FK → dealers | нет | **Только CRM/бэкенд**, не в UI |
| `registration_source` | enum | да | `qr`, `site`, `catalog` |
| `registration_qr_token` | string | нет | Первый QR |
| `registered_at` | datetime | да | |

**Правило закрепления (скрыто от пользователя):**

При регистрации, если есть `qr_token` и QR привязан к `dealer_id` — записать `dealer_id` в БД сайта и отправить в CRM webhook `user.registered`. Пользователь **не видит** сообщений вида «вы клиент дилера …».

### 3.8. Отзывы (`reviews`)

**Зачем:** отзыв после QR типа `review`; **сразу публично** на bententrade.uz; дублирование на сайт компаньона; опционально **фото**.

**Политика публикации (согласовано):**

- **Любой** отправленный отзыв **сразу показывается** на bententrade.uz (без очереди модерации).
- Фото — **опционально** (0–N файлов в форме).
- Одновременно CRM дублирует отзыв партнёру (webhook), тоже **сразу** после отправки.

| Поле | Тип | Зачем |
|------|-----|-------|
| `id` | int | PK |
| `user_id` | FK | Автор |
| `rating` | int 1–5 | Оценка |
| `text` | text | Текст |
| `status` | enum | `published` (по умолчанию); `hidden` — только ручное скрытие админом при нарушении |
| `qr_token` | string | QR отзыва |
| `qr_type` | string | `review` |
| `dealer_id` | FK | Скрыто |
| `product_id`, `profile_id`, `color_id` | string | Контекст |
| `motok_number` | string | Из QR |
| `partner_id` | FK | Компаньон |
| `partner_sync_status` | enum | `pending`, `sent`, `failed` |
| `partner_external_id` | string | ID на сайте партнёра |
| `created_at` | datetime | |

**Поток:**

1. Пользователь отправляет отзыв (текст + оценка + **фото по желанию**).
2. Сайт сохраняет, статус `published`, **сразу** показывает в блоке отзывов.
3. Webhook → CRM `POST /reviews`.
4. CRM → `POST {PARTNER_WEBHOOK}/reviews` (текст, оценка, **фото**, `motok_number`; без `batch_code`).

**Медиа (`review_media`):** `review_id`, `media_key` / url, `sort` — до N фото (например 3–5).

**Форма отзыва на сайте:**

| Поле | Обязательно |
|------|-------------|
| Оценка 1–5 | да |
| Текст | да |
| Фото | нет (0 или несколько) |

### 3.9. Каталог товаров (`products` в CRM)

**Зачем:** единственное место редактирования каталога; синхронизация на сайт.

CRM хранит полную карточку (как сейчас `products` + `product_i18n` + `media`). При save/publish:

```
POST https://bententrade.uz/api/sync/catalog
Authorization: Bearer {SITE_SYNC_SECRET}
```

Тело: товар, переводы ru/uz/en, цены, фото, `active`, категория, профиль, цвет.

Сайт применяет upsert и **не принимает** ручное редактирование каталога из `/admin` (или админка каталога отключена).

## 4. Что добавить в интерфейс CRM

### 4.1. Раздел «QR-коды» и печать

- Создание QR с выбором **`qr_type`**: отзыв / каталог.
- Привязка к мотку: профиль, цвет, вес, `motok_number`, `batch_code` (внутреннее).
- **Печать бирки** из CRM: PDF/PNG с QR, двумя вариантами дизайна под тип.
- Список: token, тип, статус, дилер, моток, дата печати.
- Карточка QR: **воронка** (см. 4.3) и полная **лента действий** с устройства.
- **Не удалять token** — только `inactive` / `revoked`.

### 4.2. Раздел «Дилеры»

- CRUD дилеров, контакты, страна, активность.
- Счётчики: сканы, регистрации, заявки, конверсия.

### 4.3. Воронка и лента действий (ключевой экран CRM)

Для каждого QR и каждого пользователя CRM показывает **полную цепочку** с устройства:

```
Отсканировали → Перешли на сайт → [Оставили отзыв | Открыли каталог]
    → Зашли на карточку товара → Выбрали цвет → Добавили в корзину
    → Начали заявку → Отправили заявку
```

**Агрегаты по QR (дашборд):**

| Этап | Метрика |
|------|---------|
| Скан | `qr_scanned` |
| Визит | `site_opened` (первая загрузка страницы после скана) |
| Отзыв | `review_submitted` (только QR типа `review`) |
| Каталог | `catalog_opened` (QR типа `catalog` или кнопка с лендинга) |
| Карточка | `product_viewed` + `product_id` |
| Корзина | `cart_item_added` + состав |
| Заявка | `order_submitted` |

**Лента событий:** хронология всех `crm_events` с `page_url`, `product_id`, `user_agent`, временем. Фильтр по QR, дилеру, пользователю, типу QR.

Данные приходят с сайта через webhook `POST /events` на **каждое** действие (раздел 7).

### 4.4. Каталог в CRM

- CRUD товаров, цен, фото, категорий, профилей, цветов.
- Кнопка «Опубликовать на сайт» → sync webhook.
- История синхронизаций (успех/ошибка).

### 4.5. Пользователи — карточка

- Имя, фамилия, телефон, страна.
- Источник: QR / сайт.
- Дилер закрепления.
- Первый `qr_token`.
- Вкладки: заявки, отзывы, корзина (снимок), события.

### 4.6. Заявки

- Колонки: source, qr_token, dealer, country.
- Назначение: дилер / Bententrade / оба.
- Уведомление дилеру (email/Telegram) — опционально в CRM.

### 4.7. Отзывы

- Список всех отзывов (все `published`).
- Просмотр текста и **фото**.
- Статус дублирования партнёру (`partner_sync_status`).
- Опционально: кнопка **«Скрыть»** (`hidden`) — только при нарушении, не премодерация.

### 4.8. Кабинет дилера (фаза 2)

- Только `dealer_id = текущий пользователь`.
- Пользователи и заявки с его QR или `users.dealer_id`.
- Без доступа к чужим QR и глобальным настройкам.

## 5. Структура страниц сайта

| URL | Страница | Назначение |
|-----|----------|------------|
| `/r/{qr_token}` | Лендинг после скана | По `qr_type`: отзыв или редирект в каталог |
| `/catalog?cat=rattan` | Каталог ротанга | Основная цель QR типа `catalog` |
| `/register` | Регистрация | Модалка; redirect после действия |
| `/login` | Вход | По телефону, без SMS |
| `/product.html?id=…` | Карточка | Как сейчас + события |
| `/cart` / checkout | Корзина и заявка | С QR/dealer в payload |
| `/account` | Личный кабинет | Профиль, заявки, отзывы, избранное — **без упоминания дилера** |
| `/review/new` | Форма отзыва | После регистрации; prefill из QR |
| `/contacts` | Контакты | Форма только для зарегистрированных |

**Редиректы после регистрации (query `?next=`):**

- `review` → форма отзыва
- `cart_add={product_id}` → добавить в корзину → каталог/корзина
- `checkout` → оформление заявки
- `contact` → форма связи

### 5.1. Лендинг `/r/{token}` по типу QR

**Общее для обоих типов:**

| Показывать | Не показывать |
|------------|----------------|
| «Настоящий ротанг Bententrade» | Имя дилера |
| Номер мотка, профиль, цвет, вес | **Номер партии (`batch_code`)** |
| | Страна дилера / «клиент дилера» |

**QR типа `review`:**

- Главная CTA: **«Оставить отзыв»**.
- Дополнительно: каталог, связь (без приоритета).

**QR типа `catalog`:**

- Сразу **редирект** на `/catalog?cat=rattan` (или лендинг с одной кнопкой «Смотреть каталог»).
- События: `qr_scanned` → `site_opened` → `catalog_opened`.

После скана всегда: webhook `qr_scanned` + `site_opened`.

### 5.2. Регистрация: лёгкая кастомизация по стране («плюшка»)

При выборе страны меняется **только визуал и подсказки**, не логика профиля и не привязка к дилеру:

| Что меняется (плюшка) | Что не меняется |
|----------------------|-----------------|
| Акцентный цвет шапки/кнопки (флаг-палитра) | Поля профиля те же |
| Маска телефона (`+998`, `+996`…) | Состав кабинета |
| Подзаголовок на языке страны | Права доступа |
| Маленький флаг рядом с селектом страны | Данные дилера |

| Страна | Маска | Подсказка (пример) |
|--------|-------|-------------------|
| UZ | `+998` | uz/ru подзаголовок |
| KG | `+996` | ru |
| KZ | `+7` | ru/kz лёгкий акцент |
| RU | `+7` | ru |
| TJ | `+992` | ru |
| OTHER | intl | ru/en |

**Обязательное поле — только телефон.**

### 5.3. Личный кабинет: визуальная «плюшка» по стране

Та же идея: **декор**, не другой продукт.

- Тонкая полоска/градиент в цветах флага в шапке кабинета.
- Иконка страны рядом с телефоном.
- Локализованные подписи разделов («Мои заявки» / «Buyurtmalarim») — опционально.

Содержимое кабинета **одинаковое**: имя, телефон, страна, заявки, отзывы, корзина. **Без дилера.**

## 6. Пользовательские сценарии

### Сценарий A — QR отзыва (`qr_type: review`)

1. Скан → `/r/{token}` → `qr_scanned`, `site_opened`.
2. Лендинг с данными мотка (без партии).
3. «Оставить отзыв» → регистрация по телефону → форма отзыва.
4. `review_submitted` → сразу на сайте и webhook в CRM → дубль партнёру.

### Сценарий B — QR каталога (`qr_type: catalog`)

1. Скан → `qr_scanned`, `site_opened`.
2. Редирект в каталог ротанга → `catalog_opened`.
3. Дальше полная лента в CRM: `product_viewed`, `color_selected`, `cart_item_added`, `checkout_started`, `order_submitted`.

### Сценарий C — Старый QR

1. Открыт `/r/old_token_2024`.
2. QR в БД, `status = active` → обычная страница.
3. Если `inactive` → сообщение «Моток подтверждён ранее», без ошибки 404.
4. Если `revoked` → «QR недействителен», контакты Bententrade.

### Сценарий D — Связаться

1. Кнопка «Задать вопрос» / «Связаться с Bententrade» → если не авторизован, регистрация по телефону.
2. WhatsApp/Telegram/звонок — **контакты Bententrade в UI** или единый номер; маршрутизация на дилера **только в CRM**, не на сайте.
3. Событие `contact_channel_clicked` с `channel` и скрытым `dealer_id` в webhook.

## 8. API и webhooks между сайтом и CRM

### 8.0. Направления запросов

| Направление | Назначение |
|-------------|------------|
| **CRM → отдаёт** (read API) | Сайт запрашивает данные QR при открытии `/r/{token}` |
| **Сайт → CRM** (webhooks) | События, регистрации, заявки, отзывы |

Env на сайте:

```
CRM_API_BASE=https://crm.example.com/api
CRM_WEBHOOK_SECRET=...
CRM_WEBHOOK_URL=https://crm.example.com/webhooks/bententrade
```

Подпись исходящих webhooks: `X-BTT-Signature: HMAC-SHA256(raw_body, CRM_WEBHOOK_SECRET)`.

---

### 8.1. Чтение QR (CRM read API, вызывает бэкенд сайта)

```
GET {CRM_API_BASE}/qr/{qr_token}
Authorization: Bearer {CRM_API_KEY}
```

**Ответ CRM (полный, для бэкенда сайта):**

```json
{
  "ok": true,
  "qr": {
    "token": "k4f82ms",
    "status": "active",
    "dealer_id": 3,
    "dealer": { "id": 3, "name": "Эмиль", "country": "KG", "phone": "...", "telegram": "...", "whatsapp": "..." },
    "country": "KG",
    "motok_number": "M-20481",
    "batch_code": "B-2026-04",
    "profile": { "id": "halfmoon_10", "name": "Полумесяц 10 мм" },
    "color": { "id": "tobacco", "name": "Tobacco" },
    "weight_kg": 25,
    "product_id": "rattan-halfmoon-10-tobacco"
  }
}
```

**Публичный ответ сайта** `GET /api/qr/{qr_token}` — без `dealer`, без `dealer_id`, **без `batch_code`**:

```json
{
  "ok": true,
  "qr": {
    "token": "k4f82ms",
    "qr_type": "review",
    "status": "active",
    "motok_number": "M-20481",
    "profile": { "id": "halfmoon_10", "name": "Полумесяц 10 мм" },
    "color": { "id": "tobacco", "name": "Tobacco" },
    "weight_kg": 25,
    "product_id": "rattan-halfmoon-10-tobacco",
    "redirect": null
  }
}
```

Для `qr_type: catalog` поле `redirect` может быть `"/catalog?cat=rattan"`.

`dealer_id` и `batch_code` — только в полном ответе CRM для бэкенда сайта, не отдаются фронту.

Возвращает сохранённые `qr_token`, `dealer_id`, `country` для текущей сессии/анонима.

### 8.3. События

```
POST /api/events
```

Тело — объект из раздела 7. Ответ: `{ "ok": true, "id": 12345 }`.  
Rate limit: например 60/мин на anonymous_id.

### 8.4. Регистрация (телефон — единственное обязательное поле)

```
POST /api/auth/register
```

```json
{
  "phone": "+996555123456",
  "first_name": "Айбек",
  "last_name": "Касымов",
  "country": "KG",
  "qr_token": "k4f82ms",
  "source": "qr",
  "current_page": "/r/k4f82ms",
  "anonymous_id": "anon_uuid"
}
```

Валидация: `phone` обязателен (мин. 7 цифр). Email, пароль и SMS-код **не используются**.

После успеха на сайте:

1. Создать сессию по `phone` (без OTP).
2. Сохранить внутреннюю атрибуцию (`dealer_id` из QR, если есть).
3. Webhook в CRM → `user.registered` (см. 8.8).

**Повторный вход** `POST /api/auth/login`:

```json
{ "phone": "+996555123456" }
```

Ответ — сессия, если номер уже зарегистрирован. SMS не отправляется.

### 8.6. Отзыв

```
POST /api/reviews
```

Требует авторизацию.

```json
{
  "rating": 5,
  "text": "Отличный ротанг",
  "photos": ["media_key_1", "media_key_2"],
  "qr_token": "k4f82ms",
  "product_id": "...",
  "profile_id": "...",
  "color_id": "..."
}
```

`photos` — массив, может быть пустым `[]`.

Ответ: `{ "ok": true, "id": 7, "status": "published" }` — отзыв **сразу** виден на сайте.

```
POST /api/contact
```

Добавить: `user_id`, `qr_token`, `dealer_id`, `country` — из сессии.

### 8.8. Webhooks: сайт → CRM (основной канал интеграции)

Базовый URL: `POST {CRM_WEBHOOK_URL}/{event}`

| Endpoint | Когда | Тело |
|----------|-------|------|
| `/events` | Любое из 13 событий (раздел 7) | `event_type`, контекст, `product_id`, … |
| `/user/registered` | Успешная регистрация | phone, first_name, last_name, country, qr_token, dealer_id, source, site_user_id |
| `/orders` | Новая заявка | Заказ + items + qr_token + dealer_id + site_user_id |
| `/reviews` | Новый отзыв | rating, text, photos, qr_token, qr_type, site_user_id; CRM дублирует партнёру |

### 8.9. Синхронизация каталога: CRM → сайт

```
POST https://bententrade.uz/api/sync/catalog
Authorization: Bearer {SITE_SYNC_SECRET}
```

События: `product.upsert`, `product.delete`, `product.unpublish` (полная карточка + i18n + media URLs).

Сайт отвечает `{ "ok": true, "synced": N }`. Каталог на сайте **только read-only** из CRM.

**Пример `POST .../user/registered`:**

```json
{
  "event": "user.registered",
  "site_user_id": 42,
  "phone": "+996555123456",
  "first_name": "Айбек",
  "last_name": "Касымов",
  "country": "KG",
  "qr_token": "k4f82ms",
  "dealer_id": 3,
  "source": "qr",
  "timestamp": "2026-07-10T12:00:00Z"
}
```

**Ответ CRM:** `200 { "ok": true, "crm_user_id": "uuid" }` — сайт может сохранить `crm_user_id` для идемпотентности.

**Повторы:** сайт хранит `webhook_outbox` (status pending/sent/failed) и ретраит при 5xx.

**Идемпотентность:** заголовок `X-BTT-Idempotency-Key: {uuid}` на каждый webhook.
