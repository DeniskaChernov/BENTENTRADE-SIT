# Bententrade CRM — полное руководство для агента / разработчика

**Для кого:** AI-агент или разработчик, который **строит только CRM** (не сайт, не Telegram-бот).  
**Версия:** 1.0 · **Дата:** 2026-07-11

## 0. Главное правило

> **CRM — единственный источник правды** по: дилерам, QR, каталогу товаров, статусам заказов (розница + опт), аналитике, печати бирок, маршрутизации заявок.

Сайт и Telegram-бот **не принимают бизнес-решения** — они показывают данные и шлют события. Всё, что «решает бизнес», живёт в CRM.

## 1. Что делать агенту CRM — и что НЕ делать

### ✅ Агент CRM ДЕЛАЕТ

| Область | Конкретно |
|---------|-----------|
| **Бэкенд CRM** | API, webhooks, бизнес-логика, PostgreSQL на Railway |
| **Админка CRM** | Дилеры, QR, печать бирок, каталог, заказы, опт, отзывы, воронка |
| **Мастер-данные** | Товары, цены, фото, QR-токены, дилеры, статусы |
| **Приём событий** | Webhooks от сайта: события, регистрации, заказы, отзывы |
| **Отдача данных** | Read API для сайта и бота (`GET /qr/{token}`, заказы пользователя) |
| **Push на сайт** | Sync каталога, смена статуса заказа → webhook на сайт |
| **Push партнёру** | Дублирование отзывов на сайт компаньона |
| **Статусы** | Розница и опт от 100 кг — меняет **только CRM** (менеджер в админке) |
| **Аналитика** | Воронка QR, лента действий, отчёты по дилерам |
| **Печать QR** | Генерация PDF/PNG бирок в интерфейсе CRM |

### ❌ Агент CRM НЕ ДЕЛАЕТ

| Не трогать | Почему |
|------------|--------|
| **Вёрстку сайта** (`*.html`, `assets/*.css`, `assets/*.js` в Bententrade-Sit) | Это репозиторий сайта, другой агент |
| **Cloudflare Worker сайта** (`worker/` в Bententrade-Sit) | Кроме согласованных endpoint'ов приёма sync — их делает агент сайта |
| **Лендинг `/r/{token}`** | Страница на сайте; CRM только отдаёт JSON по API |
| **Cookie consent, i18n сайта** | Уже на сайте |
| **Telegram-бот (диалоги)** | Отдельный сервис; бот ходит в **CRM API**, не в site_db |
| **Прямое подключение к D1 сайта** | Запрещено. Только HTTPS к публичным API сайта |
| **Показ дилера пользователю** | Ни в API для фронта, ни в текстах для сайта |
| **Показ `batch_code` на сайте** | Поле только в CRM и в полном internal API |
| **Редактирование каталога на сайте** | Сайт read-only; каталог пушится из CRM |
| **SMS / OTP** | Не в MVP. Вход по телефону без кода |
| **Удаление QR-токенов** | Только `inactive` / `revoked`, token вечный |

### ⚠️ Серая зона (согласовать с агентом сайта)

| Тема | Кто ведёт |
|------|-----------|
| Endpoint `POST bententrade.uz/api/sync/catalog` | Сайт реализует приём; CRM шлёт payload |
| Endpoint `POST bententrade.uz/api/sync/order-status` | Сайт реализует; CRM шлёт при смене статуса |
| `SITE_SYNC_SECRET`, `CRM_API_KEY` | DevOps / оба агента в env |
| Регистрация по телефону на сайте | Сайт — форма и сессия; CRM — webhook `user.registered` |

## 2. Карта экосистемы (куда что ведёт)

```
┌──────────────┐     webhooks (события, заказы, отзывы)      ┌─────────────┐
│    САЙТ      │ ───────────────────────────────────────────► │     CRM     │
│ bententrade  │                                              │  (ВЫ ЗДЕСЬ) │
│   .uz        │ ◄── read API (QR) + sync catalog/status ──── │             │
└──────────────┘                                              └──────┬──────┘
       ▲                                                             │
       │                                                             │ REST API
       │                                                             ▼
┌──────────────┐                                              ┌─────────────┐
│  TG-БОТ      │ ───────── заказы, статусы, пользователь ──► │  crm_db     │
│  (не вы)     │                                              │ PostgreSQL  │
└──────────────┘                                              └─────────────┘
```

**Что к чему приводит:**

| Действие пользователя | Кто фиксирует первым | Кто источник правды | Что видит пользователь |
|----------------------|----------------------|---------------------|------------------------|
| Скан QR | Сайт → webhook `qr_scanned` | CRM (`qr_scans`, `crm_events`) | Лендинг или каталог (сайт) |
| Регистрация | Сайт → сессия + webhook | CRM (`users`, `dealer_id` скрыто) | Кабинет Bententrade |
| Заказ из корзины | Сайт `POST /api/orders` + webhook | CRM (`orders`) | Заказ в кабинете (этапы по клику) |
| Менеджер меняет статус | **CRM админка** | CRM | Сайт обновляет после webhook |
| «Мои заказы» в боте | Бот → CRM API | CRM | Те же этапы, что на сайте |
| Публикация товара | **CRM** «Опубликовать» | CRM | Сайт после `sync/catalog` |
| Отзыв по QR | Сайт сразу публикует + webhook | CRM + партнёр webhook | Отзыв на сайте сразу |

### Фаза 1 — Дилеры + розничные заказы + статусы (MVP ядро)

**Цель:** менеджер в CRM видит заказы с сайта и меняет статусы; сайт (позже) получит обновления.

| # | Задача | Таблицы / API |
|---|--------|---------------|
| 1.1 | CRUD дилеров | `dealers` |
| 1.2 | Приём заказов с сайта | `POST /webhooks/bententrade/orders` |
| 1.3 | Модель розничного заказа | `orders`, `order_items` |
| 1.4 | Статусы розницы | `new`, `processing`, `shipped`, `delivered`, `cancelled` |
| 1.5 | Админка: список заказов, смена статуса | UI |
| 1.6 | Push статуса на сайт | `POST {SITE}/api/sync/order-status` |

**Логика статусов розницы:**

```
new ──► processing ──► shipped ──► delivered
  │         │            │
  └─────────┴────────────┴──► cancelled (в любой момент)
```

| Статус | Кто ставит | Что происходит дальше |
|--------|------------|------------------------|
| `new` | Автоматически при webhook заказа | Менеджер видит в CRM |
| `processing` | Менеджер в CRM | Webhook → сайт, кабинет: чип «В обработке» |
| `shipped` | Менеджер в CRM | Webhook → сайт, кабинет: «В пути» |
| `delivered` | Менеджер в CRM | Заказ закрыт |
| `cancelled` | Менеджер в CRM | Один чип «Отменён» в кабинете |

**UI на сайте (уже сделано, CRM не трогает):** кнопки «Отследить» нет — пользователь **кликает на карточку заказа**, снизу раскрываются чипы этапов. CRM обязан слать актуальный `status`.

**Критерий готовности фазы 1:**
- [ ] Заказ с сайта попадает в CRM через webhook
- [ ] Менеджер меняет статус в админке CRM
- [ ] CRM шлёт webhook на сайт (даже если сайт ещё не принимает — логировать)
- [ ] `public_id` заказа совпадает с сайтом (`BT-2049`)

### Фаза 2 — Каталог (мастер) + sync на сайт

**Цель:** товары редактируются **только в CRM**, сайт получает копию.

| # | Задача |
|---|--------|
| 2.1 | `products`, `product_i18n` (ru/uz/en), `media` |
| 2.2 | Админка: CRUD товаров, цены, фото, категории |
| 2.3 | Кнопка «Опубликовать на сайт» |
| 2.4 | `POST https://bententrade.uz/api/sync/catalog` с `SITE_SYNC_SECRET` |
| 2.5 | Лог sync: успех/ошибка, retry |

**Логика:** CRM save → CRM БД → webhook на сайт → сайт upsert в site_db → пользователь видит в каталоге.

**Не делать:** дублировать редактирование каталога в `/admin` сайта (это отключит агент сайта позже).

**Критерий готовности:**
- [ ] Новый товар в CRM появляется на сайте после publish
- [ ] Изменение цены в CRM обновляет сайт
- [ ] Unpublish скрывает товар на сайте

### Фаза 3 — QR + дилеры + воронка

**Цель:** печать бирок, учёт сканов, полная лента действий.

| # | Задача |
|---|--------|
| 3.1 | `qr_codes` с `qr_type`: `review` \| `catalog` |
| 3.2 | `GET /api/qr/{token}` — read API для бэкенда сайта |
| 3.3 | Печать бирки PDF/PNG в CRM |
| 3.4 | `qr_scans`, `crm_events` |
| 3.5 | Webhook `POST /webhooks/bententrade/events` (13 типов, см. ниже) |
| 3.6 | UI: воронка по QR, лента событий |
| 3.7 | Правило: token не удалять, только `inactive`/`revoked` |

**Read API — что отдавать сайту (бэкенд):**

- ✅ `token`, `qr_type`, `status`, `motok_number`, profile, color, weight, `product_id`
- ✅ `dealer_id`, `dealer{...}`, `batch_code` — **только для бэкенда сайта**, не для фронта
- ❌ Не отдавать `batch_code` в публичный JSON сайта

**Воронка (метрики в CRM):**

```
qr_scanned → site_opened → [review_submitted | catalog_opened]
  → product_viewed → color_selected → cart_item_added
  → checkout_started → order_submitted
```

**Критерий готовности:**
- [ ] CRM печатает QR с URL `bententrade.uz/r/{token}`
- [ ] Read API отвечает за < 200ms
- [ ] События с сайта пишутся в `crm_events`
- [ ] Дашборд по QR показывает конверсию

### Фаза 6 — Опт ротанга от 100 кг (`bulk_orders`)

**Отдельная сущность.** Не смешивать с `orders` (розница).

**Цепочка статусов:**

```
inquiry → quote_sent → confirmed → production → ready → shipped → delivered
                              ↘ cancelled
```

| Статус | RU | Триггер |
|--------|-----|---------|
| `inquiry` | Заявка | Форма на сайте/боте/менеджер в CRM |
| `quote_sent` | КП отправлено | Менеджер прикрепил цену |
| `confirmed` | Подтверждён | Клиент согласился, ≥ 100 кг |
| `production` | В производстве | Запуск в цех |
| `ready` | Готов к отгрузке | На складе |
| `shipped` | Отгружен | Отправлен |
| `delivered` | Доставлен | Закрыт |

**Поля `bulk_orders` (минимум):**

- `id`, `public_id` (например `BULK-1001`)
- `customer_phone`, `customer_name`, `country`
- `weight_kg`, `profile_id`, `color_id`, `comment`
- `status`, `quote_amount`, `quote_currency`
- `site_user_id`, `telegram_id` (nullable)
- `dealer_id` (nullable, скрыто)
- `created_at`, `updated_at`

**API для бота и сайта (CRM отдаёт):**

- `GET /api/users/{crm_user_id}/bulk-orders`
- `GET /api/bulk-orders/{public_id}`

**Критерий готовности:**
- [ ] Менеджер ведёт опт-заявку по всей цепочке в CRM
- [ ] Бот и сайт (когда сделают) показывают те же этапы чипами

## 5. Webhooks: что CRM принимает (сайт → CRM)

Базовый URL: `POST https://{CRM_HOST}/webhooks/bententrade/{endpoint}`

Подпись: `X-BTT-Signature: HMAC-SHA256(body, CRM_WEBHOOK_SECRET)`  
Идемпотентность: `X-BTT-Idempotency-Key: {uuid}`

| Endpoint | Когда | Что делает CRM |
|----------|-------|----------------|
| `/events` | Любое из 13 событий | INSERT `crm_events`, обновить воронку QR |
| `/user/registered` | Регистрация на сайте | UPSERT user, закрепить dealer, вернуть `crm_user_id` |
| `/orders` | Новый заказ | INSERT order + items, status=`new`, уведомить менеджера |
| `/reviews` | Новый отзыв | INSERT review, дубль партнёру |

### 13 типов событий (`event_type`)

| # | Тип | Когда на сайте |
|---|-----|----------------|
| 1 | `qr_scanned` | Открыт `/r/{token}` |
| 2 | `site_opened` | Первая загрузка после скана |
| 3 | `registration_started` | Открыта форма регистрации |
| 4 | `registration_completed` | Успешная регистрация |
| 5 | `catalog_opened` | Открыт каталог ротанга |
| 6 | `product_viewed` | Карточка товара |
| 7 | `color_selected` | Выбран цвет |
| 8 | `cart_item_added` | В корзину |
| 9 | `checkout_started` | Начало оформления |
| 10 | `order_submitted` | Заказ отправлен |
| 11 | `review_started` | Форма отзыва |
| 12 | `review_submitted` | Отзыв отправлен |
| 13 | `contact_channel_clicked` | WA / TG / звонок |

**Ответ CRM на каждый webhook:** `200 { "ok": true }` или `422` с понятной ошибкой. При 5xx сайт ретраит из outbox.

## 6. Webhooks и API: что CRM отдаёт (CRM → сайт / бот / партнёр)

### CRM → сайт

| Действие | Метод | Когда |
|----------|-------|-------|
| Sync каталога | `POST /api/sync/catalog` | Publish товара |
| Sync статуса заказа | `POST /api/sync/order-status` | Смена status в админке |
| (опционально) Sync отзыва | `POST /api/sync/review` | Если отзыв модерировали в CRM |

**Тело `order-status`:**

```json
{
  "public_id": "BT-2049",
  "status": "shipped",
  "updated_at": "2026-07-11T10:00:00Z"
}
```

### CRM → партнёр (отзывы)

`POST {PARTNER_WEBHOOK}/reviews` — rating, text, photos[], motok_number. **Без** `batch_code`, **без** `dealer_id`.

### CRM Read API (сайт и бот вызывают)

| Метод | Назначение |
|-------|------------|
| `GET /api/qr/{token}` | Данные QR для лендинга |
| `GET /api/users/{id}/orders` | Заказы пользователя |
| `GET /api/users/by-phone/{phone}` | Поиск по телефону |
| `GET /api/bulk-orders/{public_id}` | Опт-заявка |

Auth: `Authorization: Bearer {CRM_API_KEY}`

## 10. Безопасность (обязательно для агента CRM)

| Правило | Реализация |
|---------|------------|
| Проверять подпись входящих webhooks | HMAC, отклонять без подписи |
| Rate limit на `/webhooks/*` | По IP + idempotency key |
| `CRM_API_KEY` только server-to-server | Не в фронт, не в мобильный клиент |
| Не логировать полные телефоны в plain text | Маска `+998***4422` в логах |
| `dealer_id` не в публичных ответах для фронта сайта | Фильтр в read API |
| `batch_code` никогда наружу | Только internal admin CRM |
| RBAC в админке CRM | admin / manager / dealer (фаза 8) |
